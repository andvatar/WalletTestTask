# Приложение-кошелек

Это тестовое задание по разработке приложения, способного выдерживать нагрузку 1000 RPS на одну запись 

**Архитектура приложения**
1. Приложение основано на использовании LMAX Disruptor с шардированием по id кошелька. Это позволяет увеличить производительность при высокой нагрузке за счет пакетной обработки запросов и исключения конкуренции на уровне БД
- Каждый disruptor обрабатывает свой пул кошельков
- При поступлении запроса он перенаправляется в нужный disruptor в зависимости от id кошелька
- В рамках каждого disruptor все запросы обрабатываются последовательно
- Реализована пакетная обработка запросов для увеличения производительности
2. Реализовано хранение ключей идемпотентности в Redis. Также Redis используется в качестве кэша для get запросов 
3. Исключения обрабатываются глобальным Errorhandler

**Запуск приложения**

Приложение запускается в docker командой docker-compose up

Swagger - http://localhost:8080/swagger-ui.html
