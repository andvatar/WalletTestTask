**Приложение - кошелек**


**Технологический стек:**
- java 17 / Spring Boot 3
- LMAX Disruptor
- PostgreSQL
- Testcontainers


**Архитектура приложения:**

Запросы на обновление баланса направляются в буферы с шардированием по ID кошелька для исключения конкуренции между потоками

События накапливаются в буфере и сохраняются в БД одной транзакцией без in-memory aggregation для исключения некорректных списаний и обеспечения возможности аудита
Получение баланса реализовано синхронно


**Нерешенные проблемы**

- при высокой нагрузке на обновление баланса запрос баланса может возвращать данные, устаревшие на несколько десятков ms
- в запросах на изменение баланса нет ключа идемпотентности (я не успел его добавить)


Приложение запускается в docker командой docker-compose up

Swagger - http://localhost:8080/swagger-ui.html
